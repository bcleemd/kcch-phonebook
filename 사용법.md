# KCCH 내선번호부 편집기 및 배포 시스템 사용법

이 가이드는 로컬 환경에서 내선번호 데이터를 시각적으로 편리하게 편집하고, 이를 GitHub Pages(원격 저장소)에 안전하고 신속하게 배포하는 방법에 대해 설명합니다.

---

## 1. 시스템 핵심 구성

* **로컬 서버 구동**: `node server.js` 백엔드가 구동되어 로컬의 `PhoneBook.json` 파일을 가져오고 저장하며, GitHub CLI(`gh`)를 호출하여 자동 배포 파이프라인을 처리합니다.
* **시각적 트리 에디터**: 브라우저에서 `http://localhost:5050/editor_home/` 경로로 접속하여 복잡한 JSON 파일 구조를 시각적 트리 형태로 편하게 편집(추가, 수정, 삭제)합니다.
* **선별 배포 구조**: 개발 도구 소스(`editor_home/`, `server.js`, `package.json` 등)는 `.gitignore` 설정에 따라 자동으로 Git 커밋 대상에서 무시됩니다. 오직 뷰어 실행에 필요한 파일(`PhoneBook.json`, `index.html`, `style.css`, `script.js` 및 `web_home/` 하위 이중화 파일)들만 선별적으로 깃허브 원격 저장소(`bcleemd/kcch-phonebook`)에 안전하게 배포됩니다.

---

## 2. 실행 및 편집 방법

### 1) 로컬 개발 서버 실행
터미널에서 아래 명령을 실행하여 로컬 서버를 작동시킵니다.
```bash
npm run dev
```
* **동작**: Express 백엔드 서버가 포트 `5050`에서 실행됩니다.
* **에디터 주소**: [http://localhost:5050/editor_home/](http://localhost:5050/editor_home/)
* **로컬 뷰어 주소**: [http://localhost:5050/](http://localhost:5050/)

### 2) 데이터 편집
* **조회 및 이동**: 좌측 검색창에 장소명이나 사람 이름을 입력하면 검색 매칭된 모든 노드가 자동으로 펼쳐지고 노랑색으로 하이라이트되며 화면의 중심으로 스크롤 이동합니다.
* **이름/번호 수정**: 노드 우측에 표시되는 연필 아이콘(✏️)을 클릭하여 노드 명칭이나 전화번호 목록을 수정할 수 있습니다. (전화번호는 한 줄에 하나씩 입력)
* **하위 그룹/연락처 추가**: 그룹 노드(폴더 아이콘) 옆에 표시되는 `➕📁`(하위 그룹 추가) 또는 `➕📞`(연락처 추가) 버튼을 눌러 계층 구조 아래에 새로운 데이터를 자유롭게 추가할 수 있습니다.
* **노드 삭제**: 휴지통 아이콘(🗑️)을 누르면 해당 노드가 트리에서 삭제됩니다.

---

## 3. 저장 및 배포 자동화 파이프라인

데이터 수정이 끝나면 에디터 화면 우측 상단의 버튼들을 이용해 저장 및 배포를 진행합니다.

### [저장하기] 버튼 클릭
* **동작**: 수정한 내선번호 데이터를 파일 시스템의 아래 두 경로에 이중으로 동시 덮어씁니다.
  1. 프로젝트 루트의 `PhoneBook.json`
  2. `web_home/PhoneBook.json`

### [GitHub 배포] 버튼 클릭
* **동작**:
  1. 커밋 메시지를 입력할 수 있는 팝업창이 나타납니다.
  2. 배포 시작을 누르면 아래 Git 명령어가 순차적으로 실행되어 뷰어 관련 파일들만 원격 저장소(`main` 브랜치)로 즉시 배포됩니다.
     ```bash
     git add PhoneBook.json index.html style.css script.js web_home/PhoneBook.json web_home/index.html web_home/style.css web_home/script.js
     git commit -m "feat: update phonebook data"
     git push origin main
     ```
  3. 푸시가 완료되면 GitHub Pages가 작동하여 사용자들이 접속하는 뷰어 페이지가 무중단 업데이트됩니다.

---

## 4. 터미널을 통한 수동 배포 방법

에디터 웹 UI를 켜지 않고 터미널에서 직접 수동으로 배포하고 싶을 때는 아래 Git 명령어를 차례대로 실행하면 됩니다.
```bash
git add PhoneBook.json index.html style.css script.js web_home/
git commit -m "feat: manual update phonebook"
git push origin main
```
*(주의: 이 역시 `.gitignore` 정책이 정상 작동하여 백엔드 소스 등은 업로드에서 자동 배제됩니다.)*
